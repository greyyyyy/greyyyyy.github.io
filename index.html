创建hexo的过程

Hexo博客创建记录

第一步 准备

  1. 准备node.js

    安装完后会有两个组件一个是node.js本身，另一个是npm包管理器。
    可以来查看一下node 与 npm 的版本

  2. 安装cnpm
    利用npm装淘宝的源

  3. 利用cnpm安装hexo 

cnpm install -g hexo-cli

一样可以利用 

hexo -v

查看版本顺便验证是否成功

  1. 建立一个目录

    mkdir blog

    这个名字随意

  2. 初始化博客
    切换到这个目录之下，然后 

    hexo init

到这儿我们就搭建基本完成了

可以用 

hexo s

来启动，即可得到一个地址，在浏览器中打开即可看到。

然后利用 

hexo n " 你自己去的文章名字"

来创建博文
然后利用VS code打开，再用markdown语法来写即可。
可以看到我们的文章都在/_posts/目录下

再在blog目录下进行生成 

hexo g

即可
此时再启动 

hexo s

就可以看到自己的博文

第二步 部署到远端(github)

在这里，我是将其部署到GitHub上的
首先我们新建一个仓库(repository)

在repository name 处一定写的是 

用户名.github.io

emmm我也不知道为什么，是这样说的。

创建成功后

在命令行中，在blog的目录下安装一个git 部署的插件 

cnpm install --save hexo-deployer-git

成功后需要修改👇这个文件

_config.yml

在git bash 里可以直接利用
vim _config.yml
就可以打开，再在里面修改成👇

即可
在Windows中退出
先按ESC 再
:wq 退出

修改完成后，
部署到远端
hexo d
期间会要求登录GitHub账号
登陆成功后，即可
成功后，再将仓库名在浏览器中访问即可。

第三步 换主题

在GitHub中找到自己喜欢的主题，再克隆到本地，要注意的是，在克隆中可能与GitHub上写的不太一样。(至少我是/(ㄒoㄒ)/~~)

可以看到在clone那里的地址末尾是没有
.git的，应该加上。
在我之前没加的时候就一直报错，加了之后才成功了。

clone成功了，就继续打开
_config.yml文件，在里面，修改，theme
将theme改为你的主题的名字(以前默认landscape)
值得注意的是，修改完之后，还没有被部署到远端。所以，最后还要记得
hexo d
才会推到远端

以上就是整个基本操作完成啦:)

Markdown基本语法记录

一、标题

设置为标题的文字前面加 # 来表示，一定要注意的是#后是有空格的。

一级一个#最大

二级两个#稍小

三级四级以此类推，最多六个。

二、字体

  • **加粗**

    要加粗的文字左右两边分别用两个*包起来

  • *斜体*

    用一个*包起来

  • ***倾斜和加粗***

    用三个***包起来

  • ~~删除线~~

    每边用两个~~包起来

  • 下划线

<u>内容</u>

三、引用

在引用文字前加>即可；

也可以嵌套，即是引用的引用则多加一个

比如

你好

你好

并且还要注意的是引用要在开头

四、排版问题

(可能也是我没多查其他的博文，我找的博文并没有说明这一点。)

我发现，如果要实现提行写，要在后面两个以上空格加回车
空一行则会，额，空一行
并且，在分隔符的上面如果不分开，会自动变大加粗，效果有点像二级标题的样式。

然后分隔的用

👉 —

👉 —-

👉 ***

👉 ****

然后两个内容之间只能用一个空格。你就算打无数个空格，显示出来的间距也最多是一个空格。

五、代码

用反引号包含``

代码块有两个方法

  • 一个tab
    如果用tab要注意，必须与前面非代码块空一行才能使用成功
  • 三个```;代码块前后用三个反引号包含

六、图片和链接

![alt 属性文本](图片地址 "标题")
举例如下👇
![百度图标](https://www.baidu.com/img/superlogo_c4d7df0a003d3db9b65e9ef0fe6da1ec.png?where=super "百度")

链接就是没有感叹号 

[百度](www.baidu.com)

百度
