One-Hot 编码

One-Hot编码，又称为一位有效编码，主要是采用N位状态寄存器来对N个状态进行编码，每个状态都由他独立的寄存器位，并且在任意时候只有一位有效。

One-Hot编码是分类变量作为二进制向量的表示。这首先要求将分类值映射到整数值。然后，每个整数值被表示为二进制向量，除了整数的索引之外，它都是零值，它被标记为1。
实际上就是根据样本所具有的特征进行0/1赋值。
比如对hello world进行one hot编码。该句由字母和空格组成。字母的特征有26个特征，加上空格则一共有27个。

再比如若对 [“中国”,”美国”,”日本”,”美国”] 进行编码。

  • 确定要编码的对象– [“中国”,”美国”,”日本”,”美国”]
  • 确定分类变量–中国 美国 日本 共三种类别
    即是有三个样本，每个样本有三个特征，并将其转化为二进制向量。

我们首先进行特征的整数编码： 中国–0，美国–1，日本–2 ，并将特征按照从小到大排列

定义类

class Student(object):
    pass

class后面紧跟着的就是类名，即Student ，后面紧接着的(object)表示该类是从哪个类继承下来的。
如果没有合适的继承类，就用object类，这是所有类都会继承的类。

创建实例

创建实例直接example = Student(parameter)
如果在类的定义中没有绑定属性。也就是在我们定义类的时候，我们规定，所有实例都必须具有一些属性，就可以在定义类时绑定属性。
如果不绑定属性也是可以的。那么就不传入参数example = Student()
此时，可以自由定义属性 (绑定了属性也是可以自由定义，只是在于，如果了绑定了，就相当于规定了在我定义这个实例时，就必须要具有这些属性) 

>>> amy = Student()
>>> amy.name="Amy Smith"
>>> amy.name
'Amy Smith'

如果在类中绑定了属性 

class Student():
    def __init__(self,name,score):
        self.name=name
        self.score=score

注释： 在类中绑定属性，就是通过定义一个特殊的方法__init__，其第一个参数 (包括后面所有在类中定义的方法) 都是self。因为self就指向创建的实例本身。其后的参数就是我们绑定的属性。

此后在创建实例时，就要传入相应的参数。 

>>> amy = Student("Amy Smith",95)
>>> amy.name
"Amy Smith"

当然属性也不是一层不变的。
我们可以更改属性值。 

>>>amy.name="Tony"  
>>>amy.name

也就是说外部代码可以自由地修改实例的属性。

📌如果要让内部属性不被外部访问，可以把属性的名称前加两个下划线__，此时如果实例的变量名以___开头，就变成一个私有变量，只有内部可以访问，外部不能访问

注： 实际上并不是外部不能访问了，而是Python解释器将__name变成了_Student__name


可以看到通过我们定义的方法可以进行访问、修改属性。也可以看到我们仍然是可以访问私有变量的，只是其名字变了。

汉诺塔问题

问题描述

有A,B,C三个柱子，A为初始柱子，上有n个从高到低从小到大堆起的盘子。现要将A上的盘子全部移到C上，保证从始至终盘子要下大上小，请问如何移动？

解题思路

要找到一直在重复动作。
当n=1时，A -> C
当n=2时，A->B;A->C;B->C;
当n=3时，为了让C最下面是最大的，那么A的最后一个一定移给C，此时B有两个。那么B实际上就相当于当n=2时的C。也就是A一定要把2个全部给B。之后B又要把盘子全部给C。

这其实的思路实际上就是将所有看作一个整体，看作是上一个步骤来的，交换位置。

代码实现

// 汉诺塔问题
#include<stdio.h>
void hanoi(char a, char b, char c, int n);
int main(void)
{
    int n;
    char a = 'A', b = 'B', c = 'C';
    printf("ENTER:");
    scanf("%d", &n);
    hanoi(a, b, c, n);
}
void hanoi(char a, char b, char c, int n)
{

    if (n == 1)
    {
        printf("%c -> %c\n", a, c);
        return;
    }
    else
    {
        hanoi(a, c, b, n - 1);
        printf("%c -> %c\n", a, c);
        hanoi(b, a, c, n - 1);
    }
}

  • 对数组的地址+1则移向数组中的下一元素的地址，-1则移向上一个

      地址的内部表示方式是有硬件来决定的。很多计算机都是以字节编址的，这意味着对每个内存字节顺序进行编号。对于包含多个字节的数据类型，比如double类型的变量，对象的地址通常指的是其首字节的地址。
      对指针加1，等价于对指针的值加上它指向的对象的数值。 

    data+2 == &data[2]

🎏指针数组运算的总结

  1. 指针本身是有地址的，它自己的地址。
    也就是说，指针存着它所指向的值的地址，而它本身也是有地址的。
  2. 指针指向了数组后，所进行的所有运算，基本上均是其储存的值的运算，也就是所指向的值的地址的改变。它本身的地址不会改变。

🎱补充：关于常量

在函数中，使用的参数

  • 直接传递某值，不会改变调用函数的数据
  • 要通过函数修改值，则调用指针
  • 处理数组的函数，只能传递指针。
    因为这样效率更高。如果通过值向函数传递数组，那么函数中必须分配足够存放一份原数据的拷贝的储存空间，然后把原数组的所有数据复制到这个新数组中。

函数直接传递值，可以保证原始数据的完整性，因为用的是原始数据的拷贝。
所以要避免意外地不必要修改了数组的值，可以将数组的数据常量化。

那么编译器要知道它需要走几步
所以除了第一个以外，其余都需要写明。
这样编译器才知道，哦，原来走一个要COL个int啊。
如果再有，比如a[3][4][6]
那编译器就会知道，要下移一个，就需要走4个，而这四个中的每一个之间又隔了6个
所以就写为int function(int (*p) [][4][6],int rows);

🎐修饰符

这里主要提一下输出形式的问题。
①

%a.b*

a,b均为限制某的长度，表示f/d/g/s
*a是这整个字节的最小长度，如果不够，默认用空格补齐，如果在其前面加上0则以0补齐%0a.b*
**b对于不同类型有不同的意义

  • %f / %e
    • 表示小数点后几位,如果多了，则通过四舍五入来舍去。这个小数点后几位，对于科学计数法来说自然是5.36e+03,这个小数点后是2位，并且里面所有的字符都要算，所以一共有8个字节
  • %g
    • 有效数字位数，如果多了，而又要表达，除了四舍五入，还会转化为科学计数法，则此时的有效位数是指e前面的
  • %s
    • 打印的字符的最大数目，多了就舍去
  • %d
    • 数字部分有多少位，如果不够前面添零补齐，这个跟左对齐没有关系，也就是就算是左对齐仍然是前方添零补齐

②标志

  • #
    • 十六进制/八进制能够输出0x与0
    • 保证所有浮点类型都会打印出小数点，并且保证%g格式不会将尾随的零删掉
  • -
    • 左对齐，体现在如果要用空格补齐时，可以很清晰地看到是在前面补的，所以如果添加了这个，就会补在后面，保证内容一直在左边开始
  • +
    • 有符号数字前面添加正负号
  • 0
    • 补充为0而不是空格



我们从这两个例子中可以清晰地看出特别是对于%g如果位数不够会怎样，并且如何理解所有的浮点数类型都保证有小数点这句话，以及他们是不是四舍五入的方法舍去，以及什么是左对齐等

🎱补充: 可变数据类型与不可变数据类型

可变数据类型是指如果改变对象的值，其引用地址不变，其地址会扩张，但其首地址不变，并且对于相同值的对象，其地址是不一样的，实际上可以说是以变量为主，新建一个变量，就有新的对象，不管对象的值是不是一样的。
不可变数据类型则是，以对象的值为主来看，也就是如果对象的值不变，则它的地址都是一样的，只是有很多变量来引用这个相同的地址，而会有一个引用计数来记录有多少个引用指向了它；而对于改变对象的值，则相当于换了一个对象了，则会有新的地址。

👆可以看到，当值相等的时候，地址始终是相同的。
而如果一个值没有变量指向它的地址了，则会被垃圾回收器回收。再在下一次选择这个变量时，其地址会发生改变。而改变实际上还有另一个问题👇
现在对于number，在0-256(经本人试验，是到256)是属于小整数，是依赖于解释器运行环境，只要环境不终止，小整数的地址就是固定的，是不会被改变。

在这个例子我们可以看到，实际上，如果不是小整数的原因，不可变类型，如果一个值重新得到，则其地址应该是不同的。

而对于可变数据类型，比如list，以变量为主，变量变则，创建新的对象，也就是地址变，而两个变量相同，则地址同。

🎱补充:LEGB规则

python中查找变量时所按照的顺序，是按照LGEB规则来进行查找的。
首先我们应该要了解python的作用域。变量的声明是有很多个层次的。
LGEB代表的就是Local –> Enclosed –>Global –> Built-in

可以看到先要找一定是先找自己内部地在一层层往上找。这里如果内部要调用外部得，如果是闭包，则应该用nonlocal b

🎈闭包

🎏简单介绍

如果在一个内部函数里，对在外部作用域（但不是在全局作用域）的变量进行引用，那么内部函数就被认为是闭包(closure)。
这里就指明了闭包的一个特点就是外部函数得参数和变量都可以引用。本身来讲，函数执行完成时，他的储存空间是会被清理的。解释器最开始遇到函数时仅仅是为它开辟了存放函数名的空间，到真正调用时才有开辟空间。 

>>> def outer():
...     ls =[]
...     for i in range(5):
...         def inner(i):
...             return i*i
...         ls.append(inner(i))
...     return ls
...
>>> f1=outer()
>>> f2=outer()

>>> print(fi,f2,f3)

# 其实这种可以说并不是闭包了，只是在函数内都定义了一个函数。
# 闭包应当是会引用自由变量，并能够在函数的定义环境外执行的函数。
# 也就是能够被调用
# 在前面，我们说了闭包，实际上可以说是外部函数的参数和变量都保存在内部定义的一个函数中，叫做闭包。
# 而在此，f1,f2,f3并不是函数，只是列表。
# 另外，自由变量的意思就是不是定义在函数内的但能够被内部函数引用的变量

这样的引用，应该是可变数据可以直接在函数内部修改，而不可变数据应该是不能够修改的，需要global / nonlocal。这个不能修改，是指，在函数内部引用时，可以修改，只是这个修改并没有真的修改，你从函数出来，你的值还是之前的，只有可变和global/nonlocal这两种方法。

def createCounter():
    start = [0]
    def count():
        start[0] += 1
        return start[0]
    return count 

# 法二——应用生成器  
def createCounter():
    a=0
    def generator():
        while True:
            a += 1
            yield a
    def counter():
        return next(generator())
    return counter

counterA = createCounter()
print(counterA(), counterA(), counterA(), counterA(), counterA()) # 1 2 3 4 5
counterB = createCounter()
if [counterB(), counterB(), counterB(), counterB()]== [1, 2, 3, 4]:
