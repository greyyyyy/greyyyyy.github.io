🎈创建虚拟环境易于保存

创建一个虚拟环境，在里面安装库文件，可以不用担心某个项目及其关联的库文件冲突;并且当要发送给别人的时候可以轻松打包整个环境，只要python版本相同，就可以直接在虚拟环境运行，而不需要再装库。
🎱补充虚拟与正常安装区别: 我们平时直接pip install 某python库实际上是全局安装它，这通常需要有管理员权限或者以root身份安装，这个库文件对设备上的每一个用户和每一个项目都是存在的。
👇虚拟环境如何安装？
首先需要pip install virtualenv
然后virtualenv envriment-name
接着需要激活它

> cd enviroment-name
> source bin/activate

激活后，会发现环境名称出现在命令提示符前面，提示你现在在虚拟环境中。然后该怎么操作怎么操作即可。
当不再用时，通过释放命令来退出环境

> deactivate

🎈异常补充

我们应该有着一个易于捕捉异常又容易与阅读的程序代码，还应该希望能够很大程度上重用代码，所以要写像getSiteHTML这种通用函数，更是要有着周密的异常处理功能。

PART Ⅰ >> 获得 HTML

from urllib.requst import urlopen
from urllib.error import HTTPError
r = urlopen(url)

可能出现的异常

  • 网页在服务器上不存在(或者获取页面时出现错误)

  • 服务器不存在
    第一种异常出现时，程序会返回HTTP错误。HTTP错误可能是”404 Page Not Found” “500 Internal Server Error”等。所有类似情形函数都会抛出”HTTPError”异常。我们可以用以下方式处理👇 

    try:
    r = urlopen(url)
except HTTPError as e:
    print(e)

    如果程序返回HTTP错误代码，程序就会出现错误内容。

如果服务器不存在——链接打不开/url连接写错了，就会返回一个None对象,由此我们可以再增加一个判断是不是none 

if r is None:
    print("URL is not found")
else:
    # 程序继续的代码

PART Ⅱ >> bs解析过程

  • 标签对象不存在 >>> 返回None对象

  • None对象的子标签 >>> AttributeError错误 

    from urllib.request import urlopen 
from urllib.error import HTTPError 
from bs4 import BeautifulSoup
    def getHTML_title(url):
        try:
            html = urlopen(url)
        except HTTPError as e:
            return None
        try:
            obj = bs4.BeautifulSoup(html.read())
            title = obj.body.h1
        except AttributeError as e:
            return None
        return title
    title = getHTML_title(url)
    if title == None:
    # PART Ⅰ异常中的第二种/其他中全是None时。
        print("title could not found")
    else:
        print(title)

🎐 关于find()与findAll

findAll(tag, attributes, recursive, text, limit, keywords)
find(tag, attributes, recursive, text, keywords)
  • 标签参数 tag前面已经介绍过——你可以传一个标签的名称或多个标签名称组成的 Python 列表做标签参数。例如，下面的代码将返回一个包含 HTML 文档中所有标题标签的列表:
    .findAll({“h1”,”h2”,”h3”,”h4”,”h5”,”h6”})
  • 属性参数 attributes 是用一个 Python 字典封装一个标签的若干属性和对应的属性值。例 如，下面这个函数会返回 HTML 文档里红色与绿色两种颜色的 span 标签:
    .findAll(“span”, {“class”:{“green”, “red”}})
  • 递归参数 recursive 是一个布尔变量。你想抓取 HTML 文档标签结构里多少层的信息?如果 recursive 设置为 True，findAll 就会根据你的要求去查找标签参数的所有子标签，以及子 标签的子标签。如果 recursive 设置为 False，findAll 就只查找文档的一级标签。findAll 默认是支持递归查找的（recursive 默认值是 True）；一般情况下这个参数不需要设置，除 非你真正了解自己需要哪些信息，而且抓取速度非常重要，那时你可以设置递归参数。
  • 文本参数 text 有点不同，它是用标签的文本内容去匹配，而不是用标签的属性。假如我们 想查找前面网页中包含“the prince”内容的标签数量，我们可以把之前的 findAll 方法换 成下面的代码:
    nameList = bsObj.findAll(text=”the prince”) print(len(nameList))
    输出结果为“7”。
  • 范围限制参数 limit，显然只用于 findAll 方法。find 其实等价于 findAll 的 limit 等于 1 时的情形。如果你只对网页中获取的前 x 项结果感兴趣，就可以设置它。但是要注意， 这个参数设置之后，获得的前几项结果是按照网页上的顺序排序的，未必是你想要的那 前几项。
  • 关键词参数 keyword，可以让你选择那些具有指定属性的标签。例如:
    allText = bsObj.findAll(id=”text”) print(allText[0].get_text())

👆keyword的用法遇到class时，应写为.findAll(class_="name"),因为class是python的保留字，当然也可以.findAll("",{"class":"text"})，表达的意思也是查找class是text的标签

Python爬虫的学习，经过最初的实践项目入门。可以看到初步的爬虫实际上应用于提取某些信息，可以应用于下载文本/图片等；中等爬虫则是爬取网站，这个我们将用到scrapy库。scrapy库的使用，在这里简单介绍以下。scrapy框架有五个部分。Spider——初步的request、解析Response，从而得到item和新的request；Engine——中枢一般的存在。传递初步request给scheduler成为真的要网络连接的request、再将经过scheduler的真实的request传递给downloader、将downloader的response传递给spider、将spider处理出的item和新的request分别传递给Item Pipelines；Scheduler——对所用爬取请求进行调度管理；Downloader———根据请求下载网页；Item Pipelines ———以流水线方式处理spider产生的爬取项；由一组操作顺序组成，类似流水线，每一个操作是一个item pipeline类型；了能的操作包括:清理、检验、查重爬取项中的HTML数据，将数据存储到数据库中。除此以外还有两个中间键分别是位于Engine和Spider间spider middleware ——对请求和爬取项的再处理；修改、丢请、新增请求或爬取项——和位于Engine和Downloader间的downloader middleware——实现Engine/Scheduler/Downloader之间用户可配置的控制，也就是通过中间键操作，可以修改、丢弃、新增请求或响应。

🎐re库基本函数介绍

python中的正则表达式，即是引入re库

raw string类型表示正则表达式，表示为r'text',是不包含转义符的字符串
sring类型,有转义字符——即是遇到斜杠要再加一个斜杠
比较， 

r.'[0-9]\d{5}'  >>> raw string
'[0-9]\\d{5}'   >>> string

re库的主要功能函数

函数 说明
re.search() 在一个字符串中搜索匹配正则表达式的第一个位置，返回match对象
re.match() 从最开头开始匹配，也就是开头必须要匹配上，返回match对象
re.findall() 搜索字符串。以列表类型返回全不能匹配的子串
re.split() 将一个字符串按照正则表达式匹配结果进行分割，返回列表类型
re.finditer() 搜索字符串，返回一个匹配结果的跌倒类型，每个迭代元素是match对象
re.sub() 在一个字符串中替换所有匹配正则表达式的子串，返回替换的字符串 
re.search(pattern,string,flags=0)
· pattern: 正则表达式的字符串或原生字符串表示
· string: 待匹配的字符串
· flags: 正则表达式使用时的控制标记
常用标记 说明
re.I re.IGNORECASE 忽略正则表达式的大小写，也就是[A-Z]能够匹配小写字符
re.M re.MULTILINE 正则表达式中^不是本指以什么开头吗，加上这个后表示，每行以xx开头的
re.S re.DOTALL .本来应该是匹配除了换行符外的所有字符，有了这个控制符后，就是所有字符——包含换行符 

re.split(pattern,string,maxspilt=0,flags=0)
# 其他参数都一样

🎐re库面向对象写法

🏐举个例子，以下两种写法等同

>>> rst = re.search(r'[1-9]\d{5}','BIT 100081' )  
# 函数式写法:一次性操作  

---

>>> pat = re.compile(r'[1-9]\d{5}')
>>> rst = pat.search('BIT 100081')
# 面向对象用法:编译后多次操作

regex = re.compile(pattern,flags=0)

字符串或者原生字符串并不是正则表达式，它只是一种表示，这个regex才是正则表达式，代表了一组字符串。实现正则表达式和表示之间的关系。

之前介绍的普通函数都可以用,如下: 

>>> regex.match()

🎏 Lambda

lambda函数是一种匿名函数。 

lambda argument_list: expression

argument_lisy即是输入，是参数列表，可以有多种形式。

  • 变量 >>> a,b
  • 赋值的变量 >>> a=1,b=2
  • 参数
  • 空
  • …

expression输出，也可以说是，表达了将如何处理输入。 

>lambda x,y: x*y  
# 传入x,y，运算乘法

>lambda *args: sum(args) 
# 求和  

>add = lambda x,y:x+y
# 将add变为一个加法函数  >> add(5,6)

>time.sleep = lambda x:None 
# 使得这个函数不再起作用  

---
# lambda函数还可以作为剔除来用，expression是判断的式子时，表达的就是要返回是True的对象。 

>filter(lambda x: x % 3 == 0,[1,2,3])
# 将能够使三整除的选择出来  

>soup.findALL(lambda tag: len(tag.attrs)==2)
# 得到有两个属性的标签
