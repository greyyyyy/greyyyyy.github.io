实践练习 —— 获得Wiki某词条的所有词条链接，并随机进入一个链接，一直循环，知道找到一个词条页面中没有一个其他词条链接时停止。

from urllib.requst import urlopen
import bs4,datetime,random,re

random.seed(datetime.datetime.now())     

def getLinks(url):
    html = urlopen("https://en.wikipedia.org"+url)
    soup = bs4.BeautifulSoup(html)
    return soup.find("div",{"id":"bodyContent"}).finAll("a",href=re.compile("^(/wiki/)((?!:).)*$))

links = getLinks("/wiki/Kevin_Bacon")

while len(links) > 0:
    new = links[random.randint(0,len(links)-1)].attrs["href"]
    print(new)
    links = getLinks(new)

🎱在这里学到的一个新东西是random.seed

  • 为何要设定random.seed？如何设定？
    • 随机选择是一种随机数算法努力创造一种呈均匀分布且难以预测的数据序列。但是在算法的初始化都需要听过一个随机数种子也就是random.seed。完全相同的种子，每次产生相同的随机数序列。所以我们可以将随机数种子设为当下的系统时间。这样更具有随机性。
    • python的伪随机数生成器用的是梅森旋转算法。，比较消耗CPU资源，但是产生的随机数很难预测并且均匀分布。

🎈scrapy框架的相关知识补充

  • scarpy生成的调试信息是很多的，可以在scrapy项目中的setting.py文件中设置日志显示层级。
    scrapy日志有五种层级，按照范围递增顺序排列如下:
    • CRITICAL
    • ERROR
    • WARNING
    • DEBUG
    • INFO

设置哪一个等级，就会只显示这个等级及以上的内容。比如，设置为ERROR，则只显示ERROR与CRITICAL

  • 日志不仅可以显示在终端，也可以通过下面命令输出到用一个独立文件中: 

        scrapy crawl name -s LOG_FILE=wiki.log
如果目录中没有wiki.log，那么运行程序会创建一个新文件，并把所有日志保存其中。如果已存在，则添加新内容。

  • 保存所获得的信息 

    scrapy crawl name -o name.csv -t csv

scrapy crawl name -o name.jjson -t json

scrapy crawl name -o name.xml -t xml

🎈API

API——应用编程接口，为不同的应用提供了方便友好的接口，让不同的软件进行信息共享。
API与普通网址访问没有什么区别。API可以通过HTTP协议下载文件，和URL访问网址获取数据的协议一样。API请求使用很严谨的语法，其次API用JSON或XML格式表示数据，而不是HTML格式。
