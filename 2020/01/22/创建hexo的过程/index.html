创建hexo的过程

Hexo博客创建记录

第一步 准备

  1. 准备node.js

    安装完后会有两个组件一个是node.js本身，另一个是npm包管理器。
    可以来查看一下node 与 npm 的版本

  2. 安装cnpm
    利用npm装淘宝的源

  3. 利用cnpm安装hexo 

cnpm install -g hexo-cli

一样可以利用 

hexo -v

查看版本顺便验证是否成功

  1. 建立一个目录

    mkdir blog

    这个名字随意

  2. 初始化博客
    切换到这个目录之下，然后 

    hexo init

到这儿我们就搭建基本完成了

可以用 

hexo s

来启动，即可得到一个地址，在浏览器中打开即可看到。

然后利用 

hexo n " 你自己去的文章名字"

来创建博文
然后利用VS code打开，再用markdown语法来写即可。
可以看到我们的文章都在/_posts/目录下

再在blog目录下进行生成 

hexo g

即可
此时再启动 

hexo s

就可以看到自己的博文

第二步 部署到远端(github)

在这里，我是将其部署到GitHub上的
首先我们新建一个仓库(repository)

在repository name 处一定写的是 

用户名.github.io

emmm我也不知道为什么，是这样说的。

创建成功后

在命令行中，在blog的目录下安装一个git 部署的插件 

cnpm install --save hexo-deployer-git

成功后需要修改👇这个文件

_config.yml

在git bash 里可以直接利用
vim _config.yml
就可以打开，再在里面修改成👇

即可
在Windows中退出
先按ESC 再
:wq 退出

修改完成后，
部署到远端
hexo d
期间会要求登录GitHub账号
登陆成功后，即可
成功后，再将仓库名在浏览器中访问即可。

第三步 换主题

在GitHub中找到自己喜欢的主题，再克隆到本地，要注意的是，在克隆中可能与GitHub上写的不太一样。(至少我是/(ㄒoㄒ)/~~)

可以看到在clone那里的地址末尾是没有
.git的，应该加上。
在我之前没加的时候就一直报错，加了之后才成功了。

clone成功了，就继续打开
_config.yml文件，在里面，修改，theme
将theme改为你的主题的名字(以前默认landscape)
值得注意的是，修改完之后，还没有被部署到远端。所以，最后还要记得
hexo d
才会推到远端

以上就是整个基本操作完成啦:)
