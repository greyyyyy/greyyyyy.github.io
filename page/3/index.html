① iframe/frame

  • 介绍
    在HTML中有三种结构特征：树结构、层次结构、框结构。
    iframe正是框结构中的一员。每个iframe中都是一个独立的沙箱，它们拥有自己的window以及DOM。
    框结构一共有三种分别是frame、iframe、frameset；其中frameset与普通标签无区别，进行正常定位即可；而iframe和frame则相当于在一个HTML中自己又起了一个HTML是需要切换的。
    示例可看https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/iframe

  • 切换 

    <html lang="en">
<head>
<title>FrameTest</title>
</head>
<body>
<iframe src="a.html" id="frame1" name="myframe"></iframe>
</body>
</html>

    🏐前进 

    bowser.switch_to.frame(reference)

http与https的区别

Posted on 2020-01-31

🎈开头总·介绍

对于一个URL👇

http与https就是通信协议不同

🎈是否对数据加密

HTTP协议是一种使用明文数据传输的网络协议，没有对数据加密；而HTTPS是HTTP协议的升级，对数据进行了加密。可以说https的更安全；在一个网站如果是http，浏览器一般会提示不安全。

🎈对搜索🔍排名的影响

Google和百度，均对于HTTPS协议的网站采取优先排名。

requests 下载文件

requests 实际上由两个抓取，一个是Get，一个是POST，在这里先写Get,后面再继续补上。

requests.get(URL)

实质上得到的就是该网页的源代码。
当然，我们可以通过这种方法来下载某个网页里的一篇文章，即是获得HTML里某些标签的内容即可。
在这里，主要今天的重点在于如果有一个txt可以下载的网址。

import requests
res = requests.get(URL)
type(res)

这个返回的是Response对象，其中包含了Web服务器对请求做出的响应。

👆👆👆

  1. Response 对象有一个status_code 属性，可以检查她是否等于requests.codes.ok,了解下载是否成功。
  2. 检查成功的另一种方法就是在Response对象上调用raise_for_status()方法。如果下载文件出错，这将抛出异常。如果下载成功，就什么也不做。

接下来就是我得到后，将要保存。

file = open('name.txt','wb')
# 必须用"写二进制"模式打开该文件，即向函数传入字符串'wb',作为open()的第二参数。
# 即是该页面是纯文本，也需要写入二进制数据，而不是文本数据，目的是为了保存该文本的
# "Unicode编码"
for i in res.iter_content(number):
    file.write(i)

👆👆👆

  1. res.iter_content(number)
    是指在循环的每次迭代中，返回一段内容。每段内容都是bytes数据类型，你需要指定一段内容包含多少字节。也就是number处的内容
  2. 结果将返回写入字节数和最后所剩的字数，如图👇

    可以看到，最后剩余字数并不是指，没达到number所设定的就不能够写入，只是给你的提示，相当于，‘噢，你看，你没有整除成功，还有余呢’

👉 —

👉 —-

👉 ***

👉 ****

然后两个内容之间只能用一个空格。你就算打无数个空格，显示出来的间距也最多是一个空格。
