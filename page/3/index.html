def MaxAndMin(l):
    try:
        max=l[0]
        min=l[0]
        for index,ele in enumerate(l):
            if ele >= max:
                max = l[index]
            if ele <= min:
                min = l[index]
        return (min,max)
    except:
        return(None,None)

在这里用if，和elif其实都可以。if条件是等价的，是平行关系，也就是不管我前面这个条件成不成立，后面的还是要判断;而elif就是如果我前面成立了，后面就不判断了，属于备用选项，是有先后顺序的。
在这里，唯一可能的就是只有一个元素的时候，需要两个都判断，其他如果大于了max，肯定不会小于min。所以，也没有什么影响。
方法二: 冒泡排序
从前往后一个一个往后推，两个之间比较，大的往前，小的往后。第一次循环，一定完成了最大的在最左，即索引值最大，但是虽然两个之间是索引值大的数值大，但是不能保证三个之中索引值最小的就是最小。比如[4,5,3],经过两两之间比较，大的替换到前面得到的是[4,3,5]，所以会进行多次循环，一共应该进行n-1次 

def findMaxMin(l):
    n=len(l)
    try:
        n>0
        i=0 # 计数
        while i < n-1:
            j=0
            while j < n-1-i:
                if l[j] > l[j+1]:
                    mid = l[j+1]
                    l[j+1] = l[j]
                    l[j] = mid
                j += 1
            i += 1     # 循环完一次就加一 >>反映了循环次数
        return (l[0],l[-1])
    except:
        return (None,None)

🎱补充列表赋值

有三种方法，分别是

  • list1=list2
  • import copy
    • list2=copy.copy(list1)
    • list2=copy.deepcopy(list1)

deepcopy() 就是我们理想中的，我与你相等，但之后我与你无关，无论你怎么变，我都不变。
而copy() 与直接相等则会有改变。
实质上这两个改变的原因都差不多，都是如果改变内部是会跟着改变的，但如果是直接赋值，则不会。但区别在于，这两种方法的内部的定义不同。
对于a=b，是整个等于，也就是内部是指每一个元素。也就是如果a[0]=2,那么b[0]也会跟着改变.
而对于copy()它相当于是每一个元素拿出来相等。也就是它的内部应该是每一个元素的内部，也就是如果每一个元素是嵌套的，也就是对于a[0]是这种情况的👉a=[[1,2,3],1,2],此时的[1,2,3]里的每一个元素是copy的内部，也就是改变这个里面的元素，copy会跟着变，而如果改变像a[1]这样的，就不会跟着变。

以上，是我自己的理解，那么如果按照https://blog.csdn.net/u010712012/article/details/79754132 👈这位博主的模型来解释。
也就是内容是商品，而我的容器是标签的话。可以说，对于a=b，那么我整个商品贴上了标签；而对于copy(),则是，我里面的每一个元素贴上了标签。所以每个元素变，则相当于之前的整个商品都换了，而我每个元素的内部，相当于之前的修改这个商品。

🎈递归函数

递归函数，即是如下所示 

def a(n):
    if n==1:
        return n
    return n*a(n-1)


可以看到递归函数即是能够不断地调用，自己调用另一个函数。具体来说，递归函数即是那些能够在函数内部调动自身的函数。
但是我们也可以看到当我们调用1000次的时候，报错了。
这是因为在计算机中，函数调用时通过栈(stack)这种数据结构实现的，每当进入一个函数调用，栈就会加一层栈帧，每当函数返回，栈就会减一层。由于栈的大小不是无限的，所以调用次数过多，会导致栈溢出。

解决栈溢出的方法是尾递归。尾递归即是指，在函数返回时。调用本身，并且，return语句不能包含表达式。这样，编译器或者解释器就可以把尾递归做优化，使递归本身无论调用多少次，都只占用一个栈帧，不会出现栈溢出的情况。

所以👆是尾递归，但是，还是会溢出。

🎈论文的标准整体格式(按页数顺序)

  1. 封面。在封面处会有高校/机构名称、题目、姓名等基本信息
  2. 中文题目+摘要+关键词
  3. 对应的英文版题目+摘要+关键词
  4. 目录——引言、诸章、结论、主要参考文献、附录、致谢
  5. 正文
  6. 主要参考文献
  7. 附录
  8. 致谢

🎈选题

论文的选题很重要
可以从以下几个方面考虑

  • 根据自己的兴趣确定大致方向
  • 进行文献梳理。
    这个指的是要进行尽可能全面的查阅，防止自己研究主题被别人研究过了，或已有学术界认可的结论等。要尽量找到所有权威来源的相关文献，一方面可以避免出现重复问题，另一方面也可以为自己提供思路
  • 选题的范围不宜过大，也不过小。过大，容易空泛，而过小，可能细节，严谨程度要求高。

🎈标题

标题可以分为总标题和分标题(有时有副标题)
总标题也就是论文标题。
提名要求简明、具体、确切，能概括论文的特定内容，有助于选定关键词，符合编制题录、索引和检索的有关原则。
简明扼要、提纲挈领

可以看到，我们搜索关键词就可以得到相关的文献。当然这里也要注意关键词的选择。但是我们从这些标题中可以看到的便是非常的简明扼要，大多的取名模式都是基于某种方法+达到某种目的的形式
分标题，即是分段。这个主要是根据内容定，仍然是要简明扼要。

🎈摘要

摘要是文章主要内容的摘录，要求短、精、完整。字数少可几十字，多不超过三百字为宜。
摘要是对论文的内容不加注释和评论的简短陈述，要求扼要地说明研究工作的目的、研究方法和最终结论等，重点是结论，是一篇具有独立性和完整性的短文，可以引用、推广

👆这个我们可以看到，摘要中首先大概的叙述了研究的问题背景以及当前的一些方法，接着谈论到自己提出的方法，再写出结论。

👆而这个我们可以看到，他直接开始叙述是如何实现的，再给出结论。

综上: 均会给出自己研究的课题的一个大致叙述，或是方法步骤或是概述，但最终结尾都会给出结论。

🎱在这里学到的一个新东西是random.seed

  • 为何要设定random.seed？如何设定？
    • 随机选择是一种随机数算法努力创造一种呈均匀分布且难以预测的数据序列。但是在算法的初始化都需要听过一个随机数种子也就是random.seed。完全相同的种子，每次产生相同的随机数序列。所以我们可以将随机数种子设为当下的系统时间。这样更具有随机性。
    • python的伪随机数生成器用的是梅森旋转算法。，比较消耗CPU资源，但是产生的随机数很难预测并且均匀分布。

🎈SCI期刊

SCI期刊是指被SCI收录的期刊。

🎏SCI

🎐定义

  • SCI是一种科技文献检索系统，是国际公认的进行科学统计与科学评价的主要检索工具。
  • 通过统计大量的引文，然后得出某期刊某论文在某学科内的影响因子、被引频次、即时指数等量化指标来对期刊、论文等进行排行。被引频次高，说明该论文在它所研究的领域里产生了巨大的影响，被国际同行重视，学术水平高。

    🎐选入SCI系统

  • SCI对全球的自然科学刊物进行考察，凡影响因子大于某一临界值的刊物，则可以进入SCI系统。 进入SCI系统的刊物分为两类，即内圈和外圈，前者的影响因子高于后者，前者称为SCI刊物，后者称为SCIsearch刊物。

    🎐分区问题

  • SCI刊物还分为影响因子高区（大于1.105)、中区(在1.105和0.422之间）和低区（小于0.422)3类。
  • SCI期刊有两个分区规则:JCR分区和中科院分区。
    • JCR分区:
      JCR分区根据影响因子，某个学科的所有期刊都按照上一年的影响因子降序排列，然后平均4等分，分别是Q1,Q2,Q3,Q4
    • 中科院分区:
      一区刊:各类期刊三年平均影响因子的前5%，二区刊:6%-20%，三区刊:前21%-50%，四区刊:剩下的。

      🎈 刊物指标

      🎏 影响因子

      在一定时期（通常是前两年）内，某一刊物发表的论文，被已经进入SCI刊物的论文所引用的总次数，除以该刊物这一时期内的论文总数，即为该刊物的影响因子。一般来说每一年的6月份公布影响因子，影响因子并不是该年度，而是上一年度的影响因子。

      🎏 引用半衰期与被引用半衰期

  • 引用半衰期
    • 引用半衰期是指该期刊引用的全部参考文献中，较新一半是在多长一段时间内发表的。
      • 举例:《Nature》2015年的引用半衰期是6年，即2015年《Nature》发表论文的所有参考文献中，50%的参考文献是近6年内出版的。
  • 被引用半衰期
    • 显示一份期刊从当前年度向前推算引用数占截止当前年度被引用期刊的总引用数50%的年数，是测度期刊老化速度的一种指标。 半衰期通常不是针对个别文献或某一组文献，而是指某一学科或专业领域的文献总和而言的。

      🎏 及时指数

      是指一种期刊当年发表的文献在当年被引用的次数与当年的文献总数之比。此指标表示期刊论文所述的研究课题在当前的热门程度。

      🎏 被引频次

      自期刊创刊以来全部论文在某一年被引用的总次数（绝对数量指标）

      🎏 反应速率

      期刊前1年发表论文在评价当年的被引次数占前1年论文总量的比例（相对数量指标）。

🎐 关于find()与findAll

findAll(tag, attributes, recursive, text, limit, keywords)
find(tag, attributes, recursive, text, keywords)
  • 标签参数 tag前面已经介绍过——你可以传一个标签的名称或多个标签名称组成的 Python 列表做标签参数。例如，下面的代码将返回一个包含 HTML 文档中所有标题标签的列表:
    .findAll({“h1”,”h2”,”h3”,”h4”,”h5”,”h6”})
  • 属性参数 attributes 是用一个 Python 字典封装一个标签的若干属性和对应的属性值。例 如，下面这个函数会返回 HTML 文档里红色与绿色两种颜色的 span 标签:
    .findAll(“span”, {“class”:{“green”, “red”}})
  • 递归参数 recursive 是一个布尔变量。你想抓取 HTML 文档标签结构里多少层的信息?如果 recursive 设置为 True，findAll 就会根据你的要求去查找标签参数的所有子标签，以及子 标签的子标签。如果 recursive 设置为 False，findAll 就只查找文档的一级标签。findAll 默认是支持递归查找的（recursive 默认值是 True）；一般情况下这个参数不需要设置，除 非你真正了解自己需要哪些信息，而且抓取速度非常重要，那时你可以设置递归参数。
  • 文本参数 text 有点不同，它是用标签的文本内容去匹配，而不是用标签的属性。假如我们 想查找前面网页中包含“the prince”内容的标签数量，我们可以把之前的 findAll 方法换 成下面的代码:
    nameList = bsObj.findAll(text=”the prince”) print(len(nameList))
    输出结果为“7”。
  • 范围限制参数 limit，显然只用于 findAll 方法。find 其实等价于 findAll 的 limit 等于 1 时的情形。如果你只对网页中获取的前 x 项结果感兴趣，就可以设置它。但是要注意， 这个参数设置之后，获得的前几项结果是按照网页上的顺序排序的，未必是你想要的那 前几项。
  • 关键词参数 keyword，可以让你选择那些具有指定属性的标签。例如:
    allText = bsObj.findAll(id=”text”) print(allText[0].get_text())

👆keyword的用法遇到class时，应写为.findAll(class_="name"),因为class是python的保留字，当然也可以.findAll("",{"class":"text"})，表达的意思也是查找class是text的标签

🐱‍🏍重点补充——常见该异常的原因

① iframe/frame

  • 介绍
    在HTML中有三种结构特征：树结构、层次结构、框结构。
    iframe正是框结构中的一员。每个iframe中都是一个独立的沙箱，它们拥有自己的window以及DOM。
    框结构一共有三种分别是frame、iframe、frameset；其中frameset与普通标签无区别，进行正常定位即可；而iframe和frame则相当于在一个HTML中自己又起了一个HTML是需要切换的。
    示例可看https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/iframe

  • 切换 

    <html lang="en">
<head>
<title>FrameTest</title>
</head>
<body>
<iframe src="a.html" id="frame1" name="myframe"></iframe>
</body>
</html>

    🏐前进 

    bowser.switch_to.frame(reference)

    • 这个referen如果传入的是字符串则可以是iframe的id或者name也就是对应上文的myframe和frame1

    • 如果传入的是数字，则实际上就是index，索引。也就是对于整个HTML中的frame/iframe进行排序，切换到第几个iframe 突然想到一个问题，如果是一个iframe中还有一个iframe呢…所以这个应该是要切两次，这个索引也是平级之间

    • 如果这几个都不知道，还可以用过查找元素来找他 

      driver.switch_to.frame(driver.find_element_by_tag_name("iframe"))

      🏐后退
